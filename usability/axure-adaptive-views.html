Axure's Adaptive Views: Responsive Prototyping

Responsive Design hier, Responsive Design da, Responsive Design überall. Jede Webseite muss sich mittlerweile automatisch an unterschiedliche Displaygrößen anpassen. Daher ist es natürlich hilfreich, wenn man diese Anpassung schon bei Prototypen einfügen kann. Genau das funktioniert bei Axure ab Version 7.0. Wie das läuft, stell ich hier - in Text und Video - kurz vor.

Responsive Design ist schon längst nichts Neues mehr in der Webentwicklung, sondern mittlerweile Standard – sollte es zumindest sein. Damit wird aber schon nicht mehr nur die Anpassung von Seiten auf unterschiedliche Bildschirmgrößen beschrieben. Responsive Design wird zur Responsive Philosophy, was die Anpassung von Webseiten an alles Mögliche meint – nicht nur an die Größe des Bildschirms. Hier gibt’s einen interessanten Artikel dazu. Das soll an dieser Stelle aber nicht weiter vertieft werden. Hier geht’s um Axure und Responsive Prototyping. Also los!

Bekannt aus CSS

In Axure wird Prototyping für unterschiedliche Displaygrößen ‚Adaptive View‘ genannt. Die ganze Sache läuft so, dass man verschiedene Bildschirmgrößen festlegt. Diese fungieren als Grenze, ab der sich die Darstellung ändert. Wer Ahnung von CSS hat wird hier an „@media (min-width:480px)“ usw. denken. Statt mit Code regelt man das in Axure aber offensichtlich über die grafische Benutzeroberfläche. Dabei wird bei jeder festgelegten Größe automatisch das Ende der Bildschirmbreite durch Hilfslinien – sogenannte Guides – angezeigt.

Kurz zu den Guides: Es gibt in Axure zwei Arten. Hellblaue und pinke Guides. Diese können einfach per Drag & Drop aus dem Lineal am Rand der Arbeitsfläche auf eben diese gezogen werden. Bei einfachem Drag & Drop erstellt man eine hellblaue Linie, die nur auf der entsprechenden Seite erstellt wird. Bei Drag & Drop mit gedrückter STRG-Taste (CMD beim Mac) erstellt man eine pinke Linie, die global auf allen Seiten erstellt wird. Diese Guides sind extrem hilfreich, um die Grenzen des Designs zu verdeutlichen. Außerdem wirken sie „magnetisch“. Elemente lassen sich an diese andocken. Im unteren GIF sieht man, wie einfache und globale Guides aufgezogen werden.

Guides aufziehen:

Zurück zum Responsive Design. Sobald man eine neue Bildschirmgröße angibt, wird diese durch einen globalen Guide in der Workarea angezeigt. Bei Eingabe der verschiedenen Bildschirmgrößen macht es Sinn, jede Größe als Kind der vorherigen anzugeben. Dadurch werden Änderungen des Designs später von den Kindelementen automatisch übernommen. Das ist vergleichbar mit dem Verhalten von Mastern in Axure. Ändert man die Masterdatei, ändert sich das Design auf jeder Seite, auf der der Master abgelegt ist. Beim Adaptive View wird dies dann eben nur für die Kindelemente durchgeführt.

Unterschiedliche Displaygrößen einstellen

Um mit Adaptive View zu arbeiten, klickt man das Symbol direkt oben links über der Workarea. Daraufhin öffnet sich ein neues Fenster, in dem man die Displaygrößen, oder vielmehr –breiten, einstellen kann. Durch einen Klick auf das grüne Pluszeichen fügt man eine neue Breite hinzu und kann die weiteren Einstellungen für diese vornehmen. Das funktioniert über Presets oder auch manuell. Man arbeitet immer von einer Basis aus. Getreu dem Motto ‚Mobile First‘ fang ich hier - also oben im Video - vom kleinsten Screen aus an und beginne mit einer mobilen Bildschirmgröße von 360 Pixel. Die braucht man im Dialogfeld nicht angeben. Hier zieht man sich in seiner aktuellen Workarea am besten einfach einen Guide auf 360 Pixel.

Mobile First kann, muss aber nicht

Die nächste Bildschirmgröße soll 480 Pixel sein, die dritte 768 Pixel und die letzte bei 1024 Pixeln liegen. Wieviele Abstufungen dabei vorgenommen werden, bleibt jedem selbst überlassen. Manchmal werden auch nur drei Größen verwendet. Dabei entfällt beispielsweise die 480er Größe oder die 360er, wodurch 480 Pixel dann den kleinsten Screen darstellen. Das führt dann aber dazu, dass das mobile Design häufig schon zu früh beginnt und der Viewport auf Tablets zu gedrungen dargestellt wird.

Das erste Design auf der kleinsten Bildschirmgröße wird für die folgenden Screens übernommen (wenn dies in der Dialogbox ausgewählt wurde), muss dort aber noch auf die entsprechende Größe angepasst werden. Je breiter der Bildschrim, desto breiter sollte dabei auch das Design angelegt werden. Elemente einfach nur zu vergrößern ist selbstverständlich nicht die Lösung.

Das Ergebnis lässt sich schnell im Browser testen. In Axure einfach den Preview Button (ab Version 7 verfügbar) klicken und das Browserfenster verkleinern. Sobald man eine der angegebenen Screengrößen (hier: 1024px, 768px, 480px) unterschreitet, ändert sich die Darstellung entsprechend. Kleiner Tipp: Am Desktop oder Laptop ist ein linksbündiges Design nichts Ungewöhnliches. Auf Tablets und Smartphones sieht das allerdings unschön aus. Um den Inhalt zu zentrieren einfach die Einstellungen für ‚Page Align‘ im unteren Arbeitsbereich von Axure im Reiter ‚Page Style‘ ändern.

Mit Axure ab Version 7 lässt sich also auch für Responsive Design super prototypen. Das geht natürlich auch anders als hier vorgestellt, von der größten zur kleinsten Displaygröße. Die ganze Sache ist offensichtlich ziemlich einfach zu bewerkstelligen, hat im Verhältnis dazu aber einen enormen Mehrwert, da man Kunden bereits in frühen Entwicklungsphasen die unterschiedlichen Darstellungsformen präsentieren kann.
