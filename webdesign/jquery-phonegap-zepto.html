jQuery - PhoneGap - Zepto: Apps mit HTML und CSS


Mit HTML5, CSS3 und JavaScript ist man in Sachen Webdesign schon gut ausgerüstet. Wenn man dann noch jQuery und Co. von oben bis unten auswendig kennt, zaubern sich die magischsten Webseiten fast wie von selbst in den Browser. Bei nativen Apps sieht es da etwas anders aus. Für Android muss man Java können, für iOS braucht man Kenntnisse in Objective-C oder neuerdings in Swift – alles viel zu kompliziert. Dachten sich auch die Leute von Nitobi (die 2011 von Adobe aufgekauft wurden) und haben mit PhoneGap ein Framework entwickelt, mit dem man seine in HTML5, CSS3 und JavaScript geschriebenen WebApps in native Apps wandeln kann. Hab ich dann auch mal ausprobiert, nachdem mir jQuery in die Hände fiel.

jQuery ausprobiert – PhoneGap entdeckt

Neulich wollte ich mich mal in Sachen jQuery weiterbilden und hab mir den Kurs von Codecademy reingedrückt. Auch ohne großes JavaScript Know-How kommt man schnell mit jQuery klar. Für die Ahnungslosen: jQuery ist eine JavaScript-Bibliothek. Wenn man diese in seiner HTML Datei einbindet, kann man mit einfachsten Anweisungen jede Menge Dynamik in die eigene Seite bringen. Als ich den Codecademy Kurs durch hatte, wollte ich mein neues fabulöses Wissen natürlich austesten. Stichwort: Einkaufslisten-WebApp. Mit jQuery kann man nämlich ganz einfach neue Elemente wie divs in den HTML Quellcode einbinden, ohne diese im HTML reinzuschreiben. Die Idee war, dass der User etwas eingibt, einen Button klickt und das Eingegebene dann im Viewport erscheint. Wie man sich so eine digitale Einkaufsliste halt vorstellt.

Nicht jede App funktioniert als WebApp

Als WebApp ist so eine Einkaufsliste natürlich Mist. Falls der Nutzer sich nicht vorher einloggen kann, würden sich nachher ganz viele verschiedene User auf der gleichen WebApp tummeln und alle schön wild durcheinander an einer einzigen Einkaufsliste basteln. Totaler Quatsch. Das ganze irgendwie mit Logins zu lösen war mir zu blöd. Ein paar Google-Suchen später bin ich bei PhoneGap gelandet. Ein Framework um WebApps ganz einfach in native Apps zu wandeln.

Quellcode hochladen – App runterladen

Doch das allerbeste kommt noch: Zwar kann man PhoneGap schön installieren und dann irgendwie mit Eclipse und dem Android SDK undsoweiterundsofort sich seine App basteln, doch wenn man darauf keinen Bock hat, kann man einfach eine Zip-Datei bei build.phonegap.com hochladen und die fertige Apk-Datei runterladen. „Was? So einfach? Das ist doch ein Scherz!“ Nope, ist echt so. Echt easy. Zwar muss man sich vorher noch anmelden und kann als Nicht-Bezahler nur einmal eine Zip-Datei laden, allerdings funktioniert das Ganze auch im Zusammenspiel mit GitHub. So, dass man sein Repo so oft pullen kann, wie man will.

Tipps für PhoneGap Build

Trotzdem muss man einige Sachen beachten: Im Head der HTML Datei muss ein Link zum PhoneGap Framework angegeben werden: 

<script> src="cordova.js"></script>

Dass hier nicht phonegap.js oder Ähnliches steht hat damit zu tun, dass PhoneGap von Adobe an Apache gespendet wurde und Apache Projekte aus markenrechtlichen Gründen keine vorher verwendeten Produktenamen haben dürfen. Das ist hier aber weniger wichtig. Viel wichtiger ist es zu erwähnen, dass PhoneGap Probleme damit hat Farbverläufe (Gradients) einzubauen. Das scheint im ersten Moment nicht so schlimm, da man ja auch einfach seine Divs, Buttons und andere Elemente mit normalen Hintergrundfarben hinterlegen kann. Allerdings sind Gradients sehr hilfreich, will man nativ erscheinende Buttons oder Header designen. Eine Sache die etwas schade ist. Aber da momentan ja sowieso flaches Design modern ist, kann man diese Einschränkung noch verschmerzen.

jQuery vs. Zepto

Bevor es zum Fazit geht noch eine Sache: Man sollte sich darüber klar sein, dass die jQuery Bibliothek 241 KB groß ist. Das schlägt sich natürlich auf die Performance nieder. Eine Alternative ist hier die Zepto Bibliothek, die in der Basis das gleiche kann wie jQuery allerdings nur 24 KB groß, also zehnmal kleiner, ist. Und das wirkt sich natürlich enorm auf die Leistung aus. JQuery hat natürlich den Vorteil, dass die Community dahinter sehr viel größer ist. Das macht sich vor allem im Support und in den Plugins bemerkbar. Sucht man zum Beispiel bei Stackoverflow nach einer Antwort für ein Problem, findet man bezüglich jQuery jede Menge Lösungen , während die Zepto Themen im Vergleich dazu überschaubar sind. Falls man jedoch, so wie ich mit meiner Einkaufsliste, lediglich ein paar einfache Events braucht, kann man sich getrost Zepto zuwenden. Und wer jQuery kennt, der kennt auch Zepto, da Zepto auf dem jQuery API aufbaut.

Fazit

Wer Lust hat native Apps zu entwickeln, braucht keine Kenntnisse in Java, Objective-C oder Swift. HTML, CSS und JavaScript sind in Verbindung mit PhoneGap fast ebenso mächtige Technologien. Man sollte nur beachten, dass man das PhoneGap Framework im HTML-Headbereich verlinkt, möglichst keine Farbverläufe verwendet und zur besseren Performance auf leichte Bibliotheken wie zum Beispiel Zepto und nicht jQuery zurückgreift. Und wer es geschafft hat bis hierhin zu lesen, der kann sich HIER gerne mal die APK meiner Einkaufsliste runterladen.
